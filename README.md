# 项目介绍
* 原本为**纯蓝图**项目，现在正在改为**UE C++**项目，UE C++用游戏逻辑书写，蓝图用于特效显示。

# 原蓝图功能介绍
* 使⽤蓝图实现击败所有波次僵⼫或收集全部⾦币获胜机制。
* 通过AI⾏为树和⿊板实现僵⼫的巡逻、攻击、⾃动跟踪玩家以及丢失⽬标后恢复巡逻。
* 使⽤组件实现⼦弹发射与僵⼫攻击的效果。
* 在UI中实现了攻击反馈、⽣命值显⽰、暂停、重新开始等游戏逻辑。

# 现阶段改为UE C++项目功能，新加的功能
* 收集金币，金币不再一直旋转，只有当玩家接近到指定范围时金币才旋转。
* 将金币栏中中原本引擎自带的组件修改为手动创建的组件，并调整了层级结构，碰撞体作为根组件。


# 功能对比
## Actor的基本功能
### 收集金币
#### 原本实现方式
* 纯蓝图逻辑，通过触发重叠事件和cast to来实现过滤玩家和收集金币
#### 现在实现方式
* UE C++设置了拾取函数，人物过滤仍在蓝图逻辑中

<mark> 可扩展的功能: </mark>僵尸也可吃金币，吃完金币后金币数字会浮现在僵尸的头上
### 金币旋转
#### 原本实现方式
* 直接使用Rotating Movement Component
#### 现在实现方式
* 尝试了两个方案：
  * 方案一：Tick方案，每帧都进行判断金币与玩家的距离。
  * 方案二：Overlap事件，动态开启Tick，更节省性能
